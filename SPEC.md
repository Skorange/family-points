# 南瓜之家 - 设计规范 (SPEC)

> 版本：v2.2
> 最后更新：2026-03-25

---

## 0. 应用信息

- **名称**：南瓜之家（原家庭积分星球）
- **登录方式**：家庭成员选择 + 密码（无需邮箱）
- **预设成员**：
  - 家长：爸爸、妈妈、爷爷、奶奶
  - 孩子：小南瓜（姐姐）、小栗子（弟弟）

---

## 1. 角色定义

| 角色 | 权限 | 积分 |
|------|------|------|
| 家长 | 管理任务、奖励、审批兑换、管理成员、管理家庭资料 | 无积分（0分） |
| 孩子 | 做任务、兑换奖励、查看积分 | 有积分 |

## 3. 数据库设计（已确认）

### 3.1 用户表 (users)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| family_id | INT | 所属家庭 |
| username | VARCHAR(50) | 用户名 |
| email | VARCHAR(100) | 邮箱 |
| password_hash | VARCHAR(255) | 密码 |
| role | ENUM('parent', 'child') | 角色 |
| points | INT | 积分（仅孩子） |
| created_at | DATETIME | 创建时间 |

### 3.2 家庭表 (families)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| name | VARCHAR(100) | 家庭名称 |
| invite_code | VARCHAR(20) | 邀请码 |

### 3.3 任务表 (tasks)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| family_id | INT | 所属家庭 |
| title | VARCHAR(200) | 任务名称 |
| description | TEXT | 描述 |
| points | INT | 积分奖励 |
| repeat_type | ENUM('once', 'daily', 'weekly') | 重复类型 |
| created_by | INT | 创建者 |

### 3.4 积分记录表 (point_logs)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| user_id | INT | 用户 |
| task_id | INT | 关联任务 |
| amount | INT | 积分变动 |
| type | ENUM('earn', 'redeem', 'bonus', 'deduct') | 类型 |
| note | VARCHAR(255) | 备注 |
| created_at | DATETIME | 时间 |

### 3.5 奖励表 (rewards)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| family_id | INT | 所属家庭 |
| title | VARCHAR(200) | 奖励名称 |
| description | TEXT | 描述 |
| points_cost | INT | 所需积分 |
| stock | INT | 库存（-1=无限）|
| is_active | TINYINT | 是否上架 |

### 3.6 兑换记录表 (redemptions)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| user_id | INT | 兑换者 |
| reward_id | INT | 奖励 |
| points_spent | INT | 消耗积分 |
| status | ENUM('pending', 'approved', 'rejected') | 状态 |
| created_at | DATETIME | 申请时间 |
| completed_at | DATETIME | 完成时间 |

### 3.7 家庭公告表 (announcements)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| family_id | INT | 所属家庭 |
| title | VARCHAR(200) | 公告标题 |
| content | TEXT | 公告内容 |
| is_pinned | TINYINT | 是否置顶 |
| created_by | INT | 发布者 |
| created_at | DATETIME | 发布时间 |

### 3.8 家庭资料表 (family_info)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| family_id | INT | 所属家庭 |
| category | ENUM('birthday', 'document', 'event', 'note') | 分类 |
| title | VARCHAR(200) | 标题 |
| content | TEXT | 内容/日期/证件号等 |
| remind_days | INT | 提前提醒天数 |

## 5. UI 设计要求

### 5.1 配色方案
```
主色：#FF6B35 (橙色)
副色：#FF8E53 (浅橙)
背景：#F5F5F5 (浅灰)
卡片：#FFFFFF (白色)
文字：#333333 (深灰)
次要文字：#888888 (灰色)
成功：#4CAF50 (绿色)
危险：#F44336 (红色)
```

### 5.2 布局要求
- 最大宽度：480px（居中）
- 底部导航栏固定
- 内容区域可滚动
- 卡片圆角：12px
- 按钮圆角：8px

### 5.3 字体要求
- 使用系统字体
- 主标题：22px
- 副标题：16px
- 正文：14px
- 辅助文字：12px

## 8. 验收标准

1. 家长登录后不显示个人积分余额
2. 家长可创建/编辑/删除任务和奖励
3. 家长可在审批中心处理兑换申请
4. 家长可调整孩子积分（需填写原因）
5. 家长可添加/编辑/删除生日和证件
6. 爸妈可发布/编辑/删除家庭公告，孩子只读
7. 孩子可做任务获得积分
8. 孩子可申请兑换奖励，审批通过后扣除积分
9. 排行榜只显示孩子角色
10. 所有中文显示正常，无乱码
11. 移动端界面美观易用
